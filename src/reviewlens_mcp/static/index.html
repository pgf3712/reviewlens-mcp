WHAT IT SOLVES

Separate collection from interpretation.

An AI reviewer should not receive broad repository access and improvise its own workflow. ReviewLens exposes a small set of typed operations and returns structured evidence.

The model can interpret that evidence at the client layer. The server remains deterministic, testable and auditable.

ARCHITECTURE

The protocol is an adapter, not the application.

  1. 01AI hostDiscovers and invokes tools
  2. 02MCP adapterValidates the public contract
  3. 03Review serviceBuilds bounded evidence
  4. 04GitHub providerMock or live REST access

SECURITY MODEL

Repository content is data, never instruction.

01

Least privilege

No merge, approval, comment or write operation exists in the provider or MCP contract.

02

Bounded evidence

Repository names, paths, result counts, timeouts and diff sizes are validated.

03

Untrusted content

Instruction-like repository text is flagged as evidence and is never executed.

04

Explicit failure

Rate limits, transient failures and limitations remain visible to the client.
