# Security policy

## Supported version

The latest release on the default branch is supported.

## Reporting a vulnerability

Do not open a public issue containing a working exploit, credential, or private repository data. Use GitHub private vulnerability reporting when enabled. Include the affected version, impact, reproduction steps using synthetic data, and a suggested mitigation.

## Security posture

The MVP is deliberately read-only. Repository content is untrusted, credentials remain server-side, inputs and output sizes are bounded, and logs redact common credential patterns. These controls reduce risk but do not make the project production-ready.
